# TechTeam Reporting

Monthly AI adoption reporting for Synthesis Software Technologies. Standardises
license, project and assessment exports into a queryable data model and renders
a dashboard that publishes to GitHub Pages.

## Layout

```
data/
  schema.md                       Data model documentation
  load.py                         raw → standardised loader
  aliases.csv                     persistent email/handle → employee_code lookup
  <YYYY-MM>/
    raw/                          untouched monthly exports (drop in as-is)
    standardised/                 generated by load.py

dashboard/
  build.py                        regenerates dashboard.md from standardised/
  dashboard.md                    generated Quarto source (committed for CI)
  _quarto.yml                     output → ../dist/dashboard
  style.css                       Synthesis branding
```

See [data/schema.md](data/schema.md) for the data model.

## Monthly workflow

1. Drop raw exports into `data/<YYYY-MM>/raw/`:
   - `employees.csv` (or `employee.xlsx`) — HR master
   - `claude.csv`, `cursor.csv`, `chatgpt.csv`, `copilot.csv`, `gemini.csv` — tool exports
   - `projects.csv` — PS Project Tracker
   - `assessments.xlsx` — AI Maturity Self Assessment responses

2. Standardise:
   ```bash
   python data/load.py --month <YYYY-MM>
   ```
   This produces `data/<YYYY-MM>/standardised/{employees,license_allocations,unmatched,projects,project_ai_usage,assessments}.csv`.

3. Resolve unmatched rows by adding entries to [data/aliases.csv](data/aliases.csv),
   then re-run the loader.

4. Regenerate the dashboard:
   ```bash
   python dashboard/build.py --month <YYYY-MM>
   ```

5. Commit and push. CI renders and publishes automatically.

## Publishing

A push to `main` that touches `data/**` or `dashboard/**` triggers
[.github/workflows/build-dashboard.yml](.github/workflows/build-dashboard.yml).
The workflow:

1. Detects the latest snapshot under `data/`
2. Regenerates `dashboard.md` via `dashboard/build.py`
3. Renders with Quarto → `dist/dashboard/`
4. Pushes to the `gh-pages` branch at `/dashboard/` with `keep_files: true`

Live URL (once GitHub Pages is configured — see below):
**https://synthesis-labs.github.io/techteam-reporting/dashboard/dashboard.html**

### One-time GitHub Pages setup

After the first workflow run creates the `gh-pages` branch:

1. **Settings → Pages → Source** → "Deploy from a branch" → `gh-pages` / `(root)` → Save.
2. Wait ~30s; the URL above goes live.

## Local preview

Render the dashboard locally (requires [Quarto](https://quarto.org)):

```bash
cd dashboard
quarto render dashboard.md
open ../dist/dashboard/dashboard.html
```
